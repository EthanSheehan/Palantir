**1. Create `src/frontend-react/src/components/SigintDisplay.tsx`:**

Props interface:
```typescript
interface SigintDisplayProps {
  droneId: number | null;
  width: number;   // container width in px
  height: number;  // container height in px
}
```

Implementation:
- Import `ReactECharts` from `echarts-for-react`, `useRef`, `useEffect`, `useMemo` from React
- Import `useSimStore` from store
- Define constants: `FREQ_BINS = 64`, `TIME_SLOTS = 60`
- Use a `useRef<number[][]>` for a ring buffer: `Array(TIME_SLOTS).fill(null).map(() => Array(FREQ_BINS).fill(0))`
- `useEffect` with interval at 500ms (2Hz): read `useSimStore.getState()` to get current targets and UAV. For the given `droneId`, find the UAV's `sensor_contributions` on detected targets where `sensor_type === 'SIGINT'`. For each SIGINT contribution, map target ID to a pseudo-frequency bin: `(targetId * 7) % FREQ_BINS`. Set intensity = `contribution.confidence`. Fill background bins with `Math.random() * 0.08` (noise floor). Shift the ring buffer left (drop oldest column), push new column. Set state to trigger re-render.
- Use `useRef<number>` for a counter that increments to force `useMemo` recalculation
- `useMemo` computes ECharts option from the ring buffer:
  ```typescript
  const option = {
    animation: false,
    xAxis: { type: 'category', data: Array.from({length: TIME_SLOTS}, (_, i) => String(i)), show: false },
    yAxis: { type: 'category', data: Array.from({length: FREQ_BINS}, (_, i) => String(i)), show: false },
    visualMap: {
      min: 0, max: 1,
      inRange: { color: ['#000814', '#001d3d', '#003566', '#ffd60a', '#ffffff'] },
      show: false,
    },
    series: [{
      type: 'heatmap',
      data: flattenedData,  // [[timeIdx, freqIdx, intensity], ...]
      emphasis: { disabled: true },
    }],
    grid: { top: 0, bottom: 0, left: 0, right: 0 },
  };
  ```
- Flatten the ring buffer: `for (t in TIME_SLOTS) for (f in FREQ_BINS) push [t, f, buffer[t][f]]`
- Render: `<ReactECharts option={option} style={{ width, height }} opts={{ renderer: 'canvas' }} notMerge={false} theme="grid_sentinel" />`
- Cleanup: clear interval on unmount
- If `droneId === null`, render dark blue div with "NO SIGINT FEED" text

   - Inner div `transform: translateX(${-(drone.heading_deg / 360) * 360 - 100}px)` to scroll based on heading
   - Current heading indicator: small triangle marker at center, `border-bottom: 6px solid #00ff00`
   - Font: `10px monospace`, color `#00ff00`

2. **Fuel Gauge** — bottom left:
   - Container: `position: absolute; bottom: 24px; left: 8px; width: 80px`
   - Label: "FUEL" in `9px monospace`, color `#00ff00`
   - Blueprint `ProgressBar`: `value={Math.min(drone.fuel_hours / 24, 1)}`, `animate={false}`, `stripes={false}`
   - Intent: `fuel > 12 ? Intent.SUCCESS : fuel > 5 ? Intent.WARNING : Intent.DANGER`
   - Style override: `height: 6px` via className or wrapper div
   - Percentage text: `${Math.round((drone.fuel_hours / 24) * 100)}%` in `9px monospace`

3. **Sensor Status** — top right:
   - Container: `position: absolute; top: 4px; right: 8px`
   - Active sensor mode text: `sensorMode` in uppercase, color per sensor type (`EO_IR: #4A90E2`, `SAR: #7ED321`, `SIGINT: #F5A623`, `FUSION: #00ffff`)
   - Sensor quality: `"Q: ${(drone.sensor_quality * 100).toFixed(0)}%"` in `9px monospace`, color `#00ff00`
   - Fused target count: `"TGT: ${drone.fov_targets.length}"` in `9px monospace`, color `#00ff00`

4. **Threat Warning** — center overlay, conditional:

   - When triggered: red flash overlay div covering full HUD area, `background: rgba(255, 0, 0, 0.15)`, with "THREAT" text center, `font: bold 24px monospace`, color `#ff0000`, opacity pulsing via CSS animation `@keyframes threatPulse { 0%,100% { opacity: 0.3 } 50% { opacity: 1 } }` at 1s duration
   - Use inline `<style>` tag or add the keyframes to a `useEffect` that injects a `<style>` element (simpler: use inline style with `animation: 'threatPulse 1s infinite'` and define keyframes in a `<style>` tag within the component JSX)
