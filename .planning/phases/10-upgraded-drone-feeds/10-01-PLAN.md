**4. Create `src/frontend-react/src/hooks/useSensorCanvas.ts`:**

Export `useSensorCanvas(droneId: number | null, sensorMode: SensorMode, canvasRef: React.RefObject<HTMLCanvasElement | null>)`.

Structure:
- `useEffect` with `[droneId, sensorMode]` dependencies (canvasRef is stable ref)
- Inside effect: get canvas, get 2d context, set `canvas.width` / `canvas.height` to `canvas.clientWidth` / `canvas.clientHeight` (or default 400/300 if 0)
- Start rAF loop calling `renderLoop`
- `renderLoop`: read `useSimStore.getState()` for `uavs` and `targets`, find drone by `droneId`
- If no drone: call `drawNoFeed(ctx, w, h)` — black background with "NO FEED" text
- Otherwise call `drawSensorFrame(ctx, w, h, drone, targets, sensorMode)`
- Cleanup: `cancelAnimationFrame(animId)`

`drawSensorFrame` dispatches to:
- `sensorMode === 'EO_IR'`: call `drawEoIrFrame(ctx, w, h, drone, targets)` — dark green bg `#0a1a0a`, green grid `#0f2010`, targets drawn with radial gradient glow `#00ff88`, terrain noise via random-alpha `fillRect`, same `projectTarget()` logic from useDroneCam
- `sensorMode === 'SAR'`: call `drawSarFrame(ctx, w, h, drone, targets)` — dark bg `#0d0800`, scan-line effect (amber `#ffb300` horizontal stripe moving top-to-bottom using `(Date.now() / 30) % h`), target echoes as amber dots with bright center, velocity vectors as short lines from target in heading direction, scattered amber noise dots
- `sensorMode === 'FUSION'`: call `drawFusionFrame(ctx, w, h, drone, targets)` — canvas split in half using `ctx.save()/clip()/restore()`. Left half: EO_IR via `drawEoIrFrame(ctx, w/2, h, ...)`. Right half: SAR via `ctx.translate(w/2, 0)` + clip + `drawSarFrame(ctx, w/2, h, ...)`. Center divider line `#334155`.
- `sensorMode === 'SIGINT'`: draw nothing on canvas (SigintDisplay component handles this via ECharts in Plan 02). Just draw dark blue bg `#000814` with text "SIGINT — SEE WATERFALL DISPLAY".

**Target projection:** Copy `projectTarget()` from useDroneCam.ts. Modify to accept `width` and `height` params (not hardcoded WIDTH/HEIGHT). Use `drone.primary_target_id` for primary reticle (not `tracked_target_id`). Loop `drone.tracked_target_ids` for secondary dashed bounding boxes.

**HUD:** Include minimal HUD in canvas — drone ID, altitude, heading, mode text (same as useDroneCam drawHUD). Use mode-specific text colors. Include crosshair and lock box logic from useDroneCam.

**Important anti-patterns to avoid:**
- Do NOT use `useRef` for animId — use `let animId` inside the `useEffect` closure (matches existing useDroneCam pattern)
- Do NOT call `useSimStore()` as a hook subscription — use `useSimStore.getState()` inside rAF loop
- Do NOT share canvas context between slots — each `useSensorCanvas` call is independent
