## FusionBar Specification

FusionBar is an ECharts `bar` chart with `stack: 'fusion'`. It renders inside every EnemyCard that has a detected target (sensor_count > 0).

### Visual Spec

```
┌─────────────────────────────────┐
│ [EO_IR██][SAR██][SIGINT██]  78% │
│ ←── 120px ──→              ← 11px label
│ height: 12px
└─────────────────────────────────┘
```

- Stacked horizontal bar, x-axis max = 1.0
- Each sensor type gets its own series; value = max confidence for that type (0 if not contributing)
- Bar height: 12px (`barMaxWidth: 12`)
- No axis labels, no chart title, no legend — the bar is self-contained
- Tooltip: enabled on hover, `trigger: 'axis'`, `axisPointer: { type: 'shadow' }` — shows sensor type and confidence on hover
- Grid: `top: 0, bottom: 0, left: 0, right: 0, containLabel: false` (zero margin)
- Background: transparent (inherits EnemyCard background)
- Confidence label: `{(fused_confidence * 100).toFixed(0)}%`, 11px, color `#aaa` — displayed to the right of the bar
- When `sensor_count === 0`: render nothing (null return from FusionBar)
- When target is UNDETECTED: render nothing

### ECharts Option

```typescript
{
  tooltip: { trigger: 'axis', axisPointer: { type: 'shadow' } },
  xAxis: { type: 'value', max: 1.0, show: false },
  yAxis: { type: 'category', data: ['FUSION'], show: false },
  grid: { top: 0, bottom: 0, left: 0, right: 0, containLabel: false },
  series: ['EO_IR', 'SAR', 'SIGINT'].map(stype => ({
    name: stype,
    type: 'bar',
    stack: 'fusion',
    barMaxWidth: 12,
    itemStyle: { color: SENSOR_COLORS[stype] },
    data: [perType[stype] ?? 0],
  })),
}
```

## SensorBadge Specification

SensorBadge is a Blueprint `Tag` placed in the EnemyCard header row, directly after the target type Tag.

- Renders only when `sensor_count > 0`
- Text: `{N} SENSOR` (N=1) or `{N} SENSORS` (N>=2) — all-caps
- Font size: 11px via inline style
- Blueprint Tag props: `minimal`, `intent` per mapping table above
- No icon
- Placement: inline in EnemyCardInfo header row, after the existing type badge, before the state badge

## EnemyCard Modifications

Phase 1 adds two new rows to EnemyCard. The card structure becomes:

```
┌────────────────────────────────────────────────────────────┐
│ [SAM] [3 SENSORS] [DETECTED]          ID: TGT-004          │  ← Header row (existing + SensorBadge added)
│ UAV-2 PAINT  UAV-5 FOLLOW  UAV-7 FOLLOW                    │  ← Tracker tags (existing)
├────────────────────────────────────────────────────────────┤
│ [EO_IR█████][SAR███][SIGINT██]  78%                         │  ← FusionBar (NEW)
│ Contributing: UAV-2 (EO_IR), UAV-5 (SAR), UAV-7 (EO_IR)   │  ← Sensor contribution list (NEW)
├────────────────────────────────────────────────────────────┤
│ 44.12°N 28.84°E          Confidence: 78%                   │  ← Meta row (existing)
└────────────────────────────────────────────────────────────┘
```

### Sensor Contribution List

- Rendered only when `sensor_contributions.length > 0`
- Format: `UAV-{id} ({sensor_type})` per contributor, comma-separated
- Font: 12px weight 400 (Label role), color `#94a3b8` (Blueprint muted)
- Max display: 5 contributors; if more: append "…+{N} more"
- Sensor type text color: use sensor color tokens (EO_IR=`#4A90E2`, SAR=`#7ED321`, SIGINT=`#F5A623`)

## DroneCard Modifications

Phase 1 changes the tracked-target section of DroneCard from a single target display to a multi-target list.

### Before (Phase 0)

```
│ FOLLOWING TGT-004                                           │
```

### After (Phase 1)

```
│ TRACKING: TGT-004 [PRIMARY]  TGT-007  TGT-012              │
```

- Primary target: rendered with `[PRIMARY]` badge (Blueprint Tag, `Intent.WARNING`, `minimal`, 11px text)
- Primary target ID text: `#facc15` — reuses tracked-drone color token
- Secondary targets: rendered at Label (12px, weight 400, `#94a3b8`)
- When only one tracked target: no `[PRIMARY]` badge — just the target ID in `#facc15`
- When no tracked targets: show nothing (existing behavior preserved)

DroneCardDetails expanded section: when drone is expanded (tracked/selected), show full tracked targets list with each target's current fused_confidence. Format: `TGT-004 · 78%` at 12px.

## Cesium Fusion Ring Specification

Each target entity in `useCesiumTargets.ts` gains an `ellipse` property that renders a fusion ring.

### Ring Visual States

| sensor_count | Ring State | semiMajorAxis | Fill Opacity | Visible |
|-------------|-----------|---------------|--------------|---------|
| 0 | Hidden | — | 0.0 | No |
| 1 | Faint | 1500m | 0.2 | Yes |
| 2 | Medium | 2000m | 0.4 | Yes |
| 3+ | Strong | 2500m | 0.6 | Yes |

- Ring outline: always `Cesium.Color.CYAN` at 1.0 opacity when visible
- Ring fill: `Cesium.Color.CYAN.withAlpha(opacity)` where opacity = `Math.min(0.6, 0.2 * sensor_count)`
- Ring semiMajorAxis: `1000 + (sensor_count * 500)` meters
- Ring semiMinorAxis: equals semiMajorAxis (circle, not ellipse)
- Height reference: `Cesium.HeightReference.CLAMP_TO_GROUND`
- Height: 0 (ground level)
- Outline: true
- Implementation: `CallbackProperty` closures — do NOT recreate the entity per tick
- The fusion ring must not overlap the existing threat ring (threat ring uses red at fixed radius; fusion ring uses cyan at dynamic radius — visually distinct)

## Interaction Contracts

### Fusion Bar Hover

- ECharts tooltip appears on hover over FusionBar
- Tooltip content: sensor type name + confidence percentage (e.g. "EO_IR: 63%")
- No click action on FusionBar — it is read-only

### Sensor Contribution List — No Interaction

The `UAV-{id} (EO_IR)` text list is display-only. No click, no hover, no tooltip. Plain text.

### SensorBadge — No Interaction

Blueprint Tag rendered as non-interactive. No `onClick`. Not a button.

### DroneCard Tracked Targets List — Click to Select

Each target ID in the tracked-targets list is clickable:
- Click: `state.selectedTargetId = target.id` (same as clicking EnemyCard)
- Visual: cursor pointer, text underline on hover
- No other visual change to DroneCard state on target-link click

## Copywriting Contract

Phase 1 additions to the Phase 0 copywriting contract:

| Element | Copy |
|---------|------|
| SensorBadge (1 sensor) | "1 SENSOR" |
| SensorBadge (N sensors) | "{N} SENSORS" |
| Primary target badge | "PRIMARY" |
| Tracking section label | "TRACKING:" |
| Fusion bar label (no sensors) | (not rendered) |
| Contribution list prefix | "Contributing:" |
| Contribution list overflow | "…+{N} more" |
| Empty tracked targets | (no label — section hidden) |

All-caps convention applies to all sensor type labels in UI copy: "EO_IR", "SAR", "SIGINT".

Confidence display: always shown as integer percentage with `%` suffix (e.g. "78%", not "0.78" or "78.0%").

### Error States

Phase 1 introduces no new error states visible to the user. The underlying data can degrade (sensor_count drops to 0) — this is handled visually by hiding FusionBar and SensorBadge rather than showing an error. No error copy needed.

### Empty States

No new empty states. The existing Phase 0 empty state for enemies ("No hostile entities detected.") covers the case where the ENEMIES tab has no targets.

## Implementation Notes for Executor

1. **FusionBar ECharts instance at 10Hz**: EnemyCard re-renders on every WebSocket tick (10Hz). Use `useMemo` to compute the ECharts `option` object from `contributions` and `fused_confidence`. Only pass a new `option` object to `ReactECharts` when the data actually changes — prevents unnecessary ECharts internal redraws.

2. **Zustand selector for EnemyCard**: Select only the specific target's fusion fields, not the entire targets array. Pattern: `useSimStore(s => s.targets.find(t => t.id === targetId))` with `useShallow` or deep equality where needed.

3. **FusionBar zero-contribution guard**: Always check `if (!contributions || contributions.length === 0) return null` at the top of FusionBar. The backend guarantees `sensor_contributions: []` on undetected targets, but guard defensively.

4. **Cesium fusion ring — CallbackProperty scoping**: The closure in `CallbackProperty` must reference the `targets` Zustand selector, not a stale closure from entity creation. Use `useRef` to hold the latest targets array; close over the ref in the callback.

5. **Fusion ring vs threat ring**: Fusion ring is `Cesium.Color.CYAN` (dynamic, sensor-count-driven radius). Threat ring is `Cesium.Color.RED` (static, threat_range_km radius). They coexist on the same entity. Ensure the existing threat ring code is not modified by Phase 1 changes.

6. **sensor_type discriminated union**: Use `'EO_IR' | 'SAR' | 'SIGINT'` (not `string`) in `SensorContributionPayload` and in the `SENSOR_COLORS` record. This catches typos at compile time.

7. **SensorBadge placement**: Insert SensorBadge between the existing type Tag and the existing state Tag in EnemyCardInfo. Do not change the state Tag's position — it remains last in the header row.

8. **DroneCard primary target — primary_target_id vs tracked_target_ids[0]**: Use `primary_target_id` from the UAV payload, not `tracked_target_ids[0]`. The backend sets `primary_target_id` to the commanded target. `tracked_target_ids[0]` is insertion order, which may differ.

9. **SensorBadge and PRIMARY badge font size**: Both use `style={{ fontSize: 11 }}` as an inline style override on Blueprint Tag. Do not use `style={{ fontSize: 10 }}` — 10px is not part of the typography scale.
