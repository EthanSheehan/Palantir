- Sidebar drag handle: 8px wide, full height — not a content element

## Cesium Visual Spec

### Globe Base
- Tile provider: CARTO dark `https://basemaps.cartocdn.com/dark_all/{z}/{x}/{y}.png`
- `globe.baseColor`: `Cesium.Color.BLACK`
- `scene.backgroundColor`: `Cesium.Color.BLACK`
- `skyAtmosphere.hueShift`: `-0.5`
- `skyAtmosphere.brightnessShift`: `-0.8`
- `scene.fog.density`: `0.0001`
- Lighting: enabled (`globe.enableLighting = true`)
- Sun/moon: hidden
- Clock: fixed at `2023-06-21T10:00:00Z` (daytime lighting, controlled)

### UAV Entities
- Billboard: custom SVG/GLSL icon per mode (inherit from existing `drones.js`)
- Position: `SampledPositionProperty` with Hermite interpolation degree 2
- Tracking selected drone: `viewer.trackedEntity` set (third-person camera)
- Selected drone ID label: shown when tracked, yellow `#facc15` text

### Target Entities
- Billboard: type-specific icon, colored per `TARGET_MAP` from `targets.js`
- Threat ring: circle polygon drawn around SAM/RADAR types (radius from theater YAML `threat_range_km`)
- Lock ring (PAINT mode): pulsing red ring around locked target

### Zone Grid
- `GroundPrimitive` with `GeometryInstance` per zone
- Fill color: `getImbalanceColor(imbalance)` — red (shortage) to blue (surplus), 0 alpha at zero
- Border: separate `GroundPrimitive`, white at low opacity
- Grid visibility states: 0=OFF, 1=SQUARES ONLY (borders), 2=ON (fill + borders)
- Created once on first render, updated via `getGeometryInstanceAttributes` (no recreation)

### Flow Lines
- `Polyline` entities between source/target coordinates from server `flows` array
- Color: `#38bdf8` (sky blue) at 0.4 alpha
- Width: 1px

### Compass
- Custom canvas-based compass needle drawn at fixed viewport position

## Interaction Flows

### 1. Drone Selection

```
User clicks drone card (single click)
  → triggerDroneSelection(entity, 'macro')
  → Cesium camera flies to drone, macro-track mode
  → Camera controls overlay appears (top-left of CesiumContainer)
  → DroneCard expands: shows stats + mode buttons + action buttons
  → DroneCamPIP panel appears (bottom-right)

User double-clicks drone card
  → triggerDroneSelection(entity, 'thirdPerson')
  → Cesium trackedEntity set → third-person follow camera
  → Same UI state as single-click (cam PIP + expanded card)

User clicks globe/decouples
  → Camera decouple: trackedEntity cleared
  → Camera controls overlay hidden
  → DroneCamPIP hidden (or kept — see "Drone Cam Close")
  → Drone card collapses
```

### 2. Mode Commands

```
User selects drone (card expanded, mode buttons visible)

SEARCH button click:
  → sendMessage({ action: 'scan_area', drone_id })
  → No target required
  → Button border: #22c55e; active background: rgba(34,197,94,0.25)

FOLLOW / PAINT / INTERCEPT button click:
  → If no target selected: button text → "Pick target", pulse animation
  → If target selected: sendMessage({ action, drone_id, target_id })
  → Button border/color = mode color; active fill = color at 0.25 alpha

Active mode button visual: filled background at 0.25 alpha of mode color + border at full color
Inactive mode button visual: border at mode color, no fill, text at mode color
```

### 3. Target Selection

```
User clicks enemy card
  → state.selectedTargetId = target.id
  → Card gets 'selected' class (border highlight at state color)
  → Cesium camera flyTo target entity at HeadingPitchRange -45° pitch, 1000m offset

User clicks another card
  → Previous card loses 'selected' class
  → New card gains 'selected' class + camera fly

Target ID now set in store → mode command buttons with needsTarget=true become executable
```

### 4. Strike Board Approval Flow

```
PENDING entry:
  ┌─────────────────────────────────────────┐
  │ SAM            strike-abc-123  [PENDING] │  ← left border: #eab308
  │ Confidence: 85%  Priority: 9.2  ROE: ... │
  │ [APPROVE]  [REJECT]  [RETASK]            │
  └─────────────────────────────────────────┘

User clicks APPROVE:
  → sendMessage({ action: 'approve_nomination', entry_id, rationale: 'Commander approved' })
  → Entry transitions to APPROVED state
  → Left border changes to #22c55e
  → APPROVE/REJECT/RETASK buttons hidden
  → COA cards appear (from cached hitl_update payload)

COA card (APPROVED entry):
  ┌──────────────────────────────────────────┐
  │  HIMARS Rocket (SURFACE)    Pk: 87%       │
  │  TTE: 12m  Risk: 2.1  Score: 0.82        │
  │  [AUTHORIZE]  [REJECT]                   │
  └──────────────────────────────────────────┘

User clicks AUTHORIZE:
  → sendMessage({ action: 'authorize_coa', entry_id, coa_id, rationale: 'COA authorized' })
  → COA card gains 'coa-authorized' class (green tint)
  → Buttons hidden

User clicks REJECT (entry):
  → sendMessage({ action: 'reject_nomination', entry_id, rationale: 'Commander rejected' })
  → Entry left border → #ef4444, status badge → REJECTED
  → Buttons hidden, entry fades to lower visual priority

Sort order: PENDING first, then APPROVED, then RETASKED, then REJECTED
```

### 5. Drone Cam PIP

```
DroneCamPIP appears when drone selected (selectedDroneId != null)
  → Panel position: bottom-right of CesiumContainer, absolute
  → Canvas: 400×300px, fixed size
  → Header: "DRONE CAM" label + close button (×)

Canvas render loop (requestAnimationFrame):
  → Background: #141a14
  → Grid lines: #1e231e at 50px intervals
  → Target shapes drawn if in HFOV (60°) and within 15km sensor range
  → HUD overlay: drone telemetry (top-left), position (top-right)
  → Mode text: colored per mode (PAINT=#ff0000, FOLLOW=#a78bfa, etc.)
  → Crosshair: #00ff00 at center (hidden in PAINT mode — replaced by lock box)
  → Lock box: pulsing red rectangle when mode=PAINT on tracked target

Close button (×):
  → DroneCamPIP hidden
  → animationFrame NOT cancelled (resumes if drone re-selected)
  → state.droneCamVisible = false

Cleanup on unmount: cancelAnimationFrame(animFrame.current)
```

### 6. Theater Selection

```
HTMLSelect change:
  → sendMessage({ action: 'set_theater', theater: selectedValue })
  → Cesium camera flyTo theater center coordinates (from server theaterBounds)
  → Zone grid rebuilds (server sends new zones in next tick)
```

### 7. Grid Visibility Toggle

```
Button cycles through 3 states:
  State 0 (OFF):           label="Grid Visibility: OFF",       color=#e2e8f0, border=rgba(148,163,184,0.2)
  State 1 (SQUARES ONLY):  label="Grid Visibility: SQUARES ONLY", color=#a78bfa, border=rgba(167,139,250,0.5)
  State 2 (ON):            label="Grid Visibility: ON",        color=#38bdf8, border=rgba(56,189,248,0.5)

Initial state: 2 (ON)
```

### 8. Sidebar Resize

```
User mousedown on resize handle (right edge of sidebar)
  → isResizing = true, cursor = 'col-resize'
  → mousemove: sidebar.style.width = Math.min(800, Math.max(280, e.clientX - 48)) + 'px'
  → viewer.resize() called on each move event
  → mouseup: isResizing = false, cursor reset
```

## Responsive Behavior

This is a desktop C2 application. No mobile breakpoints. Minimum supported viewport: 1280×800px.

Sidebar resize constraints:
- Minimum width: 280px (prevents collapse below readable card width)
- Maximum width: 800px (prevents sidebar from dominating the globe view)
- Default width: 300px

CesiumContainer: always fills remaining viewport width. `viewer.resize()` called on every sidebar drag event to recompute canvas dimensions.

Drone cam PIP: fixed size (400×300px). Does not resize. Positioned absolute bottom-right with 16px margin from edges.

## Dark Theme Specifics

Blueprint `bp5-dark` handles:
- Card backgrounds, input backgrounds, button styles, tag colors
- All intent colors (SUCCESS=#22c55e approx, DANGER=#ef4444 approx, WARNING=#eab308 approx)
- Scrollbar styling (webkit)
- Focus ring styling (only on keyboard via FocusStyleManager.onlyShowFocusOnTabs())

Custom overrides needed (not provided by Blueprint):
- Mode badge colors (inline `style.color` + `style.borderColor`) — Blueprint Tag does not have custom color props
- Strike entry left border (inline `style.borderLeftColor`)
- Tracked drone card background tint (inline `style.background`)
- Drone cam canvas — entirely custom (2D canvas, no Blueprint)
- Demo banner — custom `background: rgba(239, 68, 68, 0.15)`, `border-bottom: 1px solid rgba(239, 68, 68, 0.4)`, text `#ef4444`, uppercase, monospace
- Grid visibility button active colors (inline style toggled on click)
- Sidebar resize handle — custom, not a Blueprint component

CSS class convention: Blueprint classes (`bp5-*`) take precedence. Custom classes are prefixed with `palantir-` to avoid collisions.

## Implementation Notes for Executor

1. **StrictMode**: Disable React StrictMode in `main.tsx`. The Cesium Viewer uses a DOM ref and will be double-created in StrictMode dev mode, producing a blank canvas. Guard with `if (viewerRef.current) return;` OR just omit `<React.StrictMode>`.

2. **Cesium hook pattern**: All Cesium hooks receive `viewerRef: RefObject<Cesium.Viewer | null>`. Guard every effect body with `if (!viewer || viewer.isDestroyed()) return;`. Never store Viewer in React state.

3. **Zone primitive**: Create `GroundPrimitive` ONCE on first render. Store in `useRef`. Update colors via `getGeometryInstanceAttributes(id)` — never destroy and recreate on each state update.

4. **DroneCam cleanup**: `useEffect` cleanup must call `cancelAnimationFrame(animFrame.current)` on unmount. Use `useRef` for the frame ID, not state.

5. **Zustand selectors at 10Hz**: Never select the entire `uavs` or `targets` array in a leaf component. Use per-item selectors (`s.uavs.find(u => u.id === id)`) and `useShallow` from Zustand for arrays in list parents.

6. **Blueprint CSS import order** in `main.tsx`:
   ```
   import '@blueprintjs/core/lib/css/blueprint.css';
   import '@blueprintjs/icons/lib/css/blueprint-icons.css';
   // custom palantir.css after
   ```

7. **vite-plugin-cesium order**: In `vite.config.ts`, `cesium()` must appear before `react()` in the plugins array.

8. **Cesium Ion token**: Move from hardcoded string in `map.js` to `import.meta.env.VITE_CESIUM_TOKEN` in `shared/constants.ts`. Add to `.env` file (gitignored).
