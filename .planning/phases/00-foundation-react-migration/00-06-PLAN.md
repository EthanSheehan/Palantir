**1. Create src/frontend-react/src/cesium/useCesiumCompass.ts:**

Port map.js `initCompass()` (lines 170-232) + `getCompassCenter()` (lines 236-246) + `initMouseTracking()` (lines 248-261).

- Creates two entities: compass needle (polyline, #facc15, width 3) and compass ring (polyline, #facc15 at 0.6 alpha, width 3, 64-segment circle, 1500m radius).
- Both use `CallbackProperty` for positions.
- Compass center: if drone tracked, use drone position at ground level; otherwise use mouse position.
- Needle direction: if tracked drone, read heading from drone entity `_lastHeading` property (requires access to droneEntitiesRef). Subtract PI from heading and apply rotation matrix to get forward vector.
- Mouse tracking: `MOUSE_MOVE` event on viewer, stores current mouse position for compass center when no drone tracked.

Takes `viewerRef` and `droneEntitiesRef` as parameters.

**2. Create src/frontend-react/src/cesium/useCesiumMacroTrack.ts:**

Port map.js `initMacroTracking()` (lines 278-300).

- Uses `preUpdate` event listener on viewer scene.
- When `macroTrackedId` is set and ready, applies camera position delta to follow drone without third-person lock.
- Reads from store: `trackedDroneId` (as macroTrackedId analogue), plus internal state for `isMacroTrackingReady` and `lastDronePosition`.
- Store these in refs, not React state.

**3. Create src/frontend-react/src/cesium/useCesiumClickHandlers.ts:**

Port mapclicks.js `initMapClickHandlers()` (all 114 lines).

Single click:
- If `isSettingWaypoint` and tracked drone: pick position, send `{ action: 'move_drone', drone_id, target_lon, target_lat }`, place waypoint, reset isSettingWaypoint.
- Otherwise: `drillPick` movement, check for `uav_` prefix → select drone (macro view with 250ms debounce), `target_` prefix → select target.

Double click:
- If UAV entity picked: select drone (thirdPerson view).
- Otherwise: spike action `{ action: 'spike', lon, lat, radius: 0.5, magnitude: 20 }` with temporary red cylinder visual (500ms).

Drone selection triggers:
- `setTrackedDrone(droneId)` in store
- `viewer.flyTo(entity, ...)` with HeadingPitchRange offset
- For macro: keep camera delta tracking. For thirdPerson: set `viewer.trackedEntity`.

**4. Create src/frontend-react/src/cesium/useCesiumRangeRings.ts:**

Port rangerings.js. Exposes `toggleRangeRings(droneId)` function.

Creates 10 concentric rings with decreasing altitude (2000m→0m) and increasing radius (0→50km). Each ring uses CallbackProperty position centered on waypoint or drone entity.

Store ring entities in a ref: `Record<number, Cesium.Entity[]>`.

**5. Create src/frontend-react/src/cesium/useCesiumWaypoints.ts:**

Port drones.js `_placeWaypoint()` (lines 293-330).

Manages waypoint cylinders (green, 2000m tall) and trajectory polylines (dashed green) per drone. Store in ref: `Record<number, { waypoint: Cesium.Entity; trajectory: Cesium.Entity }>`.

Exposes `placeWaypoint(droneId, cartesian)` function.

Waypoint visibility controlled by `showAllWaypoints` from store and current tracked drone.

**6. Create src/frontend-react/src/cesium/useCesiumLockIndicators.ts:**

Port drones.js `updateLockIndicators()` (lines 257-287).

Subscribes to `uavs` from store. For UAVs in PAINT mode with `tracked_target_id`, adds a red pulsing lock ring (ellipse, 500m radius, RED at 0.8 alpha outline) on the target entity. Removes ring when UAV exits PAINT mode.

Takes `targetEntitiesRef` as parameter.
