# 04 — UX & Workflow Analysis

## User Journey Map

### Cold Start
1. User opens localhost:3000 → Cesium globe, sidebar defaults to MISSION tab
2. WebSocket connects to :8000/ws, sends IDENTIFY + subscribe
3. Backend delivers first state tick, theater/UAVs/targets populate
4. **No onboarding modal, tooltip tour, or help text** — new user sees "System Dashboard" with no guidance

### Full Manual F2T2EA Cycle

| Phase | User Action | UI Element | Gap |
|-------|------------|------------|-----|
| Find | Watch globe for detected targets | Cesium globe, EnemiesTab | No alert when new target appears |
| Fix | Monitor VerificationStepper in ENEMIES tab | VerificationStepper, FusionBar | No audio/visual alert on state progression |
| Track | Click drone → select target → FOLLOW/PAINT | DroneModeButtons (hidden behind card toggle) | Two-step interaction hidden behind expansion |
| Target | Wait for NOMINATED, see Strike Board | StrikeBoardEntry APPROVE/REJECT/RETASK | Strike board buried at bottom of MISSION scroll |
| Engage | Approve → COA appears → AUTHORIZE | StrikeBoardCoa | Rationale hardcoded "Commander approved" — no text input |
| Assess | Switch to ASSESS tab | AssessmentTab | No timestamps, corridor details are just counts |

### Demo / Autopilot Mode
- Only indication: 32px red DemoBanner at top
- AutonomyToggle can still be manipulated — no lockout or warning
- **No visual narrative of current F2T2EA phase**
- **No pause/resume from UI**
- **No AI reasoning chain visible** — only terminal output in AssistantWidget

### Autonomy Level Switching
- Three-segment control: MANUAL → SUPERVISED → AUTONOMOUS
- **No confirmation dialog before AUTONOMOUS** — one mis-click removes human from lethal loop
- TransitionToast (10s countdown) only visible on ASSETS tab — invisible from other tabs
- No system-wide notification overlay for pending transitions

## Missing Visualizations (Critical for C2)

- **No system-wide alert center** — critical events only in scrollable logs
- **No F2T2EA kill chain progress indicator** — no persistent phase display
- **No timeline/replay** — all events live, no playback
- **No BDA summary panel** — BDA mode exists but no results display
- **No map click context menu** — right-click on target/drone does nothing
- **No AI decision explanation panel** — scores shown but no narrative
- **No target history/track** — neutralized targets fade, no kill log
- **No target velocity/speed display**
- **Threat range rings only for SAM/MANPADS** — ARTILLERY/APC have data but no rings

## Missing Output Formats

- No export to PDF/JSON/CSV
- No SITREP generation button (agent exists but no UI trigger)
- No mission debrief view
- No screenshot/clip function
- CommandLog ring buffer (200 events) — older events lost, no export
- No COA comparison table view

## Error Handling Issues

| Scenario | Problem |
|----------|---------|
| Connection loss | No explanation, no retry count, no "last connected" |
| Theater switch failure | `console.error` — silent swallow, dropdown changes but theater doesn't |
| No target selected for mode | Subtle 600ms pulse + brief label change — easy to miss |
| Range/Detail buttons | `onClick={() => {}}` — dead buttons, zero feedback |
| No COAs loaded | COA section simply doesn't render — no loading state |

## Power User Acceleration (Top 10)

1. **Global command palette (Cmd+K)** — type commands instead of click through tabs
2. **Strike board as floating overlay** — always visible, not buried in scroll
3. **Click-to-assign on globe** — target right-click → "Follow with UAV-X"
4. **Batch approve/reject** for multiple nominations
5. **Escape key → force MANUAL** — hardware-brakeable safety override
6. **Sticky selected target** — globe fly-to on sidebar selection
7. **ISR Queue one-click dispatch** — recommended UAVs shown but no action button
8. **Coverage gap → send drone** — gap shown but no dispatch action
9. **Drone fuel system-wide warning** — no cross-tab alert on low fuel
10. **Theater switch confirmation** on live mission

## Keyboard Shortcuts — Gaps

| Exists | Missing |
|--------|---------|
| 1-6 for map modes | Tab to cycle sidebar tabs |
| Click drone/target on globe | A/R to approve/reject focused nomination |
| Double-click drone (3rd person) | Space to pause/resume |
| Double-click map (spike) | Escape for MANUAL / deselect / exit waypoint |
| — | W for waypoint mode |
| — | Arrow keys for camera pan |

## Multi-Monitor / Responsive

- Fixed-pane split layout (sidebar 280-800px + globe remainder)
- No multi-window support — C2 operators need separate monitors for globe, panels, feeds
- QUAD PIP (801×601) overflows viewport on screens < 1400px
- Font sizes 9-11px hardcoded — unusable at standard viewing distances
- PIP position hardcoded (bottom: 16, right: 16) — overlaps CameraPresets on small screens

## AI Decision Transparency — Missing

- No explanation of WHY a COA was ranked above alternatives
- No identification of WHICH agent generated each recommendation
- No confidence trajectory (sparkline/history)
- No sensor attribution for state transitions
- Rejected action rationale always hardcoded
- Swarm coordinator reasoning invisible

## Audit Trail — Gaps

- CommandLog ring buffer (200 events) fills in minutes
- No persistent audit log in UI (JSONL exists on backend)
- No filtering by drone, target, or action type
- Rationale always hardcoded "Commander approved/rejected"
- No operator ID field for COA authorization
- Transition approvals show action names only — no context

## Onboarding — Nonexistent

- No welcome/tutorial modal
- No tooltips on any control
- No help button or documentation link
- No guided first-mission walkthrough
- No map legend (color coding for types, modes, zones)
- No glossary for military acronyms (ISR, ROE, COA, F2T2EA, TEL, MANPADS, BDA)
- Autonomy toggle has no description of each level

## Top Friction Points by Severity

### Critical
1. No confirmation before AUTONOMOUS — one mis-click removes human from lethal loop
2. Transition toasts tab-local — operator misses 10-second approval windows
3. Strike board buried in scroll — PENDING nominations not visually prominent

### High
4. No AI decision explanation — operator can't understand recommendations
5. ISR Queue has no dispatch action — shows needs but requires 3+ manual steps
6. Dead buttons (Range, Detail) create false affordance
7. No alert for new high-priority events
8. TheaterSelector errors silently

### Medium
9. No keyboard shortcuts beyond map modes
10. QUAD PIP overflows viewport
11. No fuel/RTB system-wide alert
12. No export/report capability
13. No onboarding
